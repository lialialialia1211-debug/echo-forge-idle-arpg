# 基礎數值

日期：2026-07-01
狀態：Design Contract v0.1
目的：先定義戰鬥陀螺的基礎屬性、先後順序與 POE 三維對照，作為後續 engine、data、save、UI 優化依據。

## 1. 核心設計原則

戰鬥陀螺不是一般角色站樁戰鬥。陀螺必須持續旋轉，旋轉代表還活著，旋轉停止就等於倒下。

因此本遊戲應把「陀螺還在轉」和「陀螺本體耐打」分成兩套概念：

| 概念 | 中文名稱 | 定義 | 失敗條件 |
| --- | --- | --- | --- |
| Spin Energy | 旋能 | 陀螺維持旋轉的能量。旋能越低，實際轉速越低 | 旋能歸零，陀螺倒下 |
| Structure Integrity | 結構耐久 | 陀螺本體被撞擊、切削、破壞的承受力 | 耐久歸零，陀螺破壞 |
| Current RPM | 目前轉速 | 戰鬥中即時旋轉速度，受旋能、質量、體積、摩擦、失衡影響 | 不直接死亡，但會降低攻擊與技能頻率 |

目前程式裡的 `spinIntegrity` 比較像「結構耐久」與「戰鬥生命」混合。後續若要更接近戰鬥陀螺，建議逐步拆成：

```text
/-
spinEnergy      旋能
structure       結構耐久
currentRpm      目前轉速
maxRpm          最大轉速
```

短期可以先保留 `spinIntegrity`，但文件與 UI 應開始把語意拆清楚。

## 2. 三維主屬性

對照 Path of Exile 的力量、敏捷、智慧，本遊戲建議使用自己的三維名稱。

| POE 屬性 | 本遊戲三維 | 核心幻想 | 主要影響 |
| --- | --- | --- | --- |
| 力量 Strength | 鐵重 | 重、硬、撞擊壓制 | 質量、結構耐久、撞擊、護甲、防失衡 |
| 敏捷 Dexterity | 疾旋 | 快、準、切入角度 | 最大轉速、移動、追蹤、刃口、暴擊 |
| 智慧 Intelligence | 共鳴 | 能量、技能、符文控制 | 旋能、能量護盾、技能頻率、冷卻、符文效率 |

三維不是最終戰鬥公式，而是決定裝備需求、天賦方向、流派傾向的上層屬性。

| 三維 | 代表流派 | 正面傾向 | 風險 |
| --- | --- | --- | --- |
| 鐵重 | 重型坦撞、Boss 壓制 | 不容易被撞飛，撞擊有力，耐久高 | 移動慢，轉向慢，轉速較低 |
| 疾旋 | 高速彈射、暴擊連撞 | 頻率高，切削強，機動好 | 容易被重型撞飛，旋能消耗快 |
| 共鳴 | 技能流、元素流、符文流 | 技能強，冷卻快，能量護盾佳 | 本體碰撞弱，需要能量管理 |

## 3. 屬性先後順序

後續所有公式應遵守這個順序，避免 UI 或單一屬性直接硬改戰鬥結果。

```text
三維主屬性
-> 零件基礎值
-> 物理派生值
-> 進場發射狀態
-> 戰鬥中旋能
-> 戰鬥中目前 RPM
-> 移動 / 碰撞 / 技能頻率
-> 傷害 / 失衡 / 彈飛 / 倒下
```

詳細說明：

| 順序 | 層級 | 說明 |
| ---: | --- | --- |
| 1 | 三維主屬性 | 鐵重、疾旋、共鳴決定流派方向 |
| 2 | 零件基礎值 | 核心、攻擊環、配重盤、軸尖、發射器等提供基礎性能 |
| 3 | 物理派生值 | 質量、體積、慣性、抓地、重心穩定 |
| 4 | 進場發射狀態 | 初始轉速、初始旋能、入場速度 |
| 5 | 戰鬥中旋能 | 被摩擦、碰撞、技能消耗影響 |
| 6 | 戰鬥中目前 RPM | 由最大 RPM 與旋能狀態推導 |
| 7 | 行為公式 | 移動速度、碰撞力、攻擊頻率、技能頻率 |
| 8 | 結果 | 傷害、失衡、彈飛、倒下、破壞 |

## 4. 基礎屬性定義

| 屬性 | 中文名稱 | 類型 | 定義 |
| --- | --- | --- | --- |
| `spinEnergy` | 旋能 | 建議新增 | 維持旋轉的能源。越低，轉速與輸出越低 |
| `structure` | 結構耐久 | 建議拆分 | 陀螺本體承受破壞的生命值 |
| `maxRpm` | 最大轉速 | 建議拆分 | 此配置理論可達到的最高轉速 |
| `currentRpm` | 目前轉速 | 戰鬥狀態 | 戰鬥中即時轉速，不應只等於面板 RPM |
| `launchPower` | 發射力 | 建議新增 | 決定進場初始轉速與初始旋能 |
| `mass` | 質量 | 已存在 | 抗擊退、撞擊力、穩定性、移動遲鈍 |
| `size` / `radius` | 體積 / 半徑 | 建議正式化 | 攻擊範圍、碰撞範圍、受擊面積、慣性 |
| `momentOfInertia` | 轉動慣量 | 建議派生 | 由質量與體積推導，影響加速與轉速維持 |
| `grip` | 抓地 | 已存在 | 抗滑、抗失衡、中心低坡抵抗 |
| `tracking` | 追蹤 | 已存在 | 主動靠近敵人的能力與命中穩定 |
| `impact` | 撞擊 | 已存在 | 碰撞傷害基礎 |
| `edge` | 刃口 | 已存在 | 切削、暴擊、火花、擦撞收益 |
| `fracture` | 破裂倍率 | 已存在 | 暴擊或破壞倍率 |
| `resonance` | 共鳴 | 已存在 | 技能能量、冷卻、恢復、符文效率 |
| `fluxGuard` | 通量護盾 | 已存在 | 類似能量護盾，偏共鳴流 |

## 5. 質量、體積、轉速的關係

質量與體積必須分開。大不一定重，小也可以很重。

| 屬性 | 正面效果 | 負面效果 |
| --- | --- | --- |
| 質量 `mass` | 撞擊更有力，不容易被撞飛，spin loss 較低，wobble 較低 | 移動速度慢，加速慢，轉向慢，最大轉速下降 |
| 體積 `size/radius` | 攻擊範圍大，碰撞範圍大，更容易撞到敵人 | 受擊面積大，轉動慣量高，最大轉速下降 |
| 轉速 `rpm` | 攻擊頻率高，技能頻率高，切削與火花更強 | 旋能消耗快，輕型高速更容易被彈飛 |
| 抓地 `grip` | 穩定，不容易滑走，不容易失衡 | 太高會降低漂移與彈射刺激 |

### 建議公式方向

初始進場轉速不應由角色等級直接決定。它應主要由發射器決定，核心與軸尖輔助，質量與體積扣速。

```text
initialRpm =
  launcher.launchPower
  + core.launchEfficiency
  + tip.spinEfficiency
  + dexterityLikeBonus
  + affixBonus
  - massPenalty
  - sizePenalty
  - inertiaPenalty
```

目前戰鬥中轉速應由最大轉速與旋能比例推導：

```text
currentRpm =
  maxRpm
  * spinEnergyRatio
  * wobblePenalty
  * frictionPenalty
  * phaseOrBuffMultiplier
```

攻擊與技能頻率應吃 `currentRpm`，不是只吃面板 `rpm`。

```text
attackFrequency = baseAttackRate * currentRpmFactor
skillFrequency  = baseCooldown / currentRpmFactor / resonanceFactor
```

旋能消耗可以由摩擦、碰撞與技能消耗組成：

```text
spinEnergyDrain =
  baseFriction
  + speedFriction
  + collisionLoss
  + skillCost
  - resonanceRecovery
```

## 6. 進場轉速來源

等級不應直接給進場轉速。等級只應間接解鎖更高階零件、詞綴、天賦。

| 來源 | 對進場狀態的影響 |
| --- | --- |
| 發射器 Launcher | 最主要來源。決定初始 RPM、初始旋能、入場速度 |
| 核心 Core | 決定最大旋能、穩定性、技能能源 |
| 軸尖 Tip | 決定摩擦、抓地、旋能消耗 |
| 攻擊環 Attack Ring | 增加攻擊範圍、刃口、撞擊，但可能增加體積與慣性 |
| 配重盤 Weight Disk | 增加質量、穩定與撞擊，但降低轉速與機動 |
| 三維屬性 | 鐵重增加質量收益，疾旋增加發射與轉速，共鳴增加旋能與技能效率 |
| 等級 | 只做間接成長，不直接灌轉速 |

正式規則：

```text
發射器決定開場
核心決定能源
軸尖決定耗能
質量決定撞擊與抗擊退
體積決定攻擊範圍與慣性
RPM 決定攻擊與技能頻率
旋能決定陀螺還能不能活著
```

## 7. 現有程式對應

目前已存在並可延伸的屬性：

| 現有屬性 | 現況 | 後續定位 |
| --- | --- | --- |
| `mass` | 已影響碰撞、抗擊退、spin loss、wobble、技能擊退 | 升級為核心物理屬性，UI 必須顯示 |
| `rpm` | 已影響技能與部分碰撞表面速度 | 拆成 `maxRpm` 與 `currentRpm` |
| `spinIntegrity` | 目前作為主要生命值 | 建議語意改為 `structure` 或拆出 `spinEnergy` |
| `guard` | 降低撞擊傷害 | 歸屬鐵重 |
| `grip` | 影響穩定與摩擦 | 同時受鐵重與軸尖影響 |
| `tracking` | 影響追擊與命中 | 主要歸屬疾旋，可被共鳴技能使用 |
| `impact` | 撞擊輸出 | 歸屬鐵重 |
| `edge` | 刃口、暴擊、擦撞收益 | 歸屬疾旋 |
| `fracture` | 暴擊倍率 | 高價輸出屬性 |
| `resonance` | 技能與恢復效率 | 歸屬共鳴 |
| `fluxGuard` | 類能量護盾 | 歸屬共鳴 |

## 8. 珍貴度排序

基礎屬性珍貴度由低至高：

| 等級 | 類型 | 屬性 |
| --- | --- | --- |
| T1 | 基礎常見 | 結構耐久、護甲、防禦、漂移、追蹤 |
| T2 | 主要戰鬥 | 撞擊、轉速、抓地、元素抗性 |
| T3 | 三維主屬性 | 鐵重、疾旋、共鳴 |
| T4 | 高價物理 | 質量、體積、刃口、彈出壓力、失衡 |
| T5 | 高價技能 | 旋能、能量護盾、冷卻恢復、保留效率 |
| T6 | 稀有輸出 | 破裂倍率、穿透、more damage、extra as |
| T7 | 掉落經濟 | 零件數量、零件稀有度 |
| T8 | 流派核心 | 額外投射、連鎖、重複施放、Boss phase bonus、Breach reward、特殊碰撞效果 |

`mass` 不應是普通小屬性。它同時影響碰撞、彈射、抗擊退、spin loss、wobble，應至少歸類為 T4 高價物理。

## 9. 後續落地順序

不得直接先改 UI 或硬改數字。後續若要正式改 engine，建議順序：

```text
system contract
-> data schema
-> engine tests
-> engine implementation
-> save/migration
-> UI connection
-> browser playtest
```

第一階段建議先做：

1. 新增 `spinEnergy` 與 `launchPower` 型別。
2. 將 `rpm` 分成面板最大轉速與戰鬥中目前轉速。
3. 將 `radius/size` 從 runtime 呈現值提升為裝備可影響的正式屬性。
4. UI 將 `mass`、`currentRpm`、`spinEnergy` 放入 Build Summary。
5. 補測試：重型陀螺更難被撞飛、輕型陀螺攻擊頻率更高、旋能低時技能頻率下降。
