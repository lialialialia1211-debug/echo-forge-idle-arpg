# Codex 開工指示 — 資訊減量 + 漸進揭露（回應真實玩家測試反饋）

> 版本 2026-07-03 ｜ 執行者：Codex ｜ 規劃/品管：Claude（驗收 Fable 5）
>
> **玩家測試反饋（原話）：「太雜、太亂、整個畫面太多資訊、根本不知道要幹嘛、一下塞太多資訊反而讓人不想玩。」**
>
> 稽核實證（commit `e7e6405`，171 tests 綠）：新手開局戰鬥畫面同時有 **約 12 個常駐區塊、80–100 個數字**，其中 telemetry-grid、run-review-strip、inspection-grid 兩張卡、event-feed 五塊佔六成資訊量——全部是 docs/13 §3.5 明文要移出戰鬥畫面、積欠至今的項目。天賦頁一進去就是 98 節點全樹、技能頁一次亮出 31 張還沒拿到的符文卡、地圖一進去畫 31 個節點只為了第一章的 4 個。同一件事（撿到裝備）被四個系統各講一次。
>
> **這批目標：戰鬥畫面瘦到只剩「打鬥＋目標＋資源」；所有進階系統改為「第一次需要時才出現」；重複資訊只留一個出口。深度不刪、只是換出場時機。**

## 先讀
- docs/13 §3.5（戰鬥畫面應保留清單——本批終於執行它）、docs/20（教學觸發器，本批的解鎖鑰匙）。
- 行號以 commit `e7e6405` 為準。

## 鐵則
1. **不刪功能，只搬位置或延後出現**——所有被移走的面板都要有新家，被隱藏的系統都有明確解鎖條件。
2. 顯示字串走 zh-Hant；解鎖判定進 selector 不散落 JSX。
3. 交付前 typecheck/test/build 全過＋刪存檔真機走新手前十分鐘。
4. 有設計疑問回報。

## 核心機制：解鎖規則（本批地基，先做）
新增 selector `isFeatureUnlocked(featureId)`，規則：**`seenTutorialIds 含對應教學步驟 id`（存檔已持久化，docs/20 D2）`|| 該步驟的觸發條件當下為真`**。教學觸發的那一刻＝介面解鎖的那一刻，兩系統天然同步、免改 schema。未解鎖的元素**完全不渲染**（不是 disabled 灰化——灰化仍是資訊）。

（開局可見分頁只剩「裝備」「背包」——新手前五分鐘的世界就是：打、撿、穿。）

### B1 地圖網絡戰爭迷霧
network 16 節點（`:2142-2178` 一帶）只渲染：**已解鎖節點＋前置已達成的下一層節點**；其餘畫成無名「？」節點（灰色小圓，無文字無數值），不可點。atlas 圖同理：`availableAtlasPoints === 0` 且無已配點時整張圖收合為一行摘要「圖集點數 0——完成網絡節點後獲得」。

### B2 天賦樹迷霧
98 節點只完整顯示：**已配點節點＋可配（前置達成）節點＋再往外一步的節點**（名稱/花費可見）；更遠的節點降為輪廓態（30% 透明度、無名稱、hover 無反應），cluster 分區底色與 keystone 剪影保留（讓玩家看得到「樹很大」的野心，但不用讀 98 個名字）。配點推進時迷霧即時後退。世界座標/不重疊測試不受影響（只改渲染態，不動座標）。

### B3 符文庫「持有優先」
技能頁符文庫（31 張全列）改為：預設只列**已持有**的符文；底部一行「圖鑑（N/31）」按鈕展開完整目錄（展開態為既有樣式）。開局零持有時顯示空狀態文案「符文會從戰鬥中掉落」。

## Task C — 注意力去重（同一件事只講一次）

1. **NextActionStrip 防抖**：非危急情境（danger/defeat/runtimeError 除外）切換建議時，最短停留 **4 秒**（常數進 balanceConfig 或 UI 常數檔）——修掉「玩家讀教學卡 3 秒、抬頭建議已換掉」的追不上感。
2. **掉落瞬間的四重奏減為二重奏**：toast（發生了什麼）＋NextActionStrip（該做什麼）保留；run-review 掉落計數已在 A1 刪除；`tut_first_drop` 教學卡**延後到玩家第一次暫停或該場結束時**才出（不跟 toast 同幀搶注意力）——教學卡觸發器普遍加一條規則：`running === true` 時入佇列不彈出，暫停/結算時才依序彈。
3. **stage-header 作為唯一進度出口**：完整度%、目標名稱只在此出現（A1 已刪 hud-grid 重複項）。

**Task C 驗收**：
- [ ] 戰鬥中掉落發生：畫面上只有 toast＋strip 兩處反應；暫停時教學卡才出。
- [ ] strip 非危急切換間隔 ≥4s（測試或手動計時）。
