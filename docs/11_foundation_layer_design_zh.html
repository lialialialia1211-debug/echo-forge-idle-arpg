📐 底層規格

0. 設計定位：分層合併

一句話：POE2 是骨架（刷寶湊 build、加法桶/乘法桶/稀有度/門檻），戰鬥陀螺是皮肉（把那套骨架用「旋轉、撞擊、能量」的物理型態演出來）。兩層各自完整，只在一個點咬合：肉體層算出的傷害種子與頻率，餵進 build 層的傷害乘算鏈。
flowchart LR subgraph BODY[肉體層 · 物理身體] F[力量 F] --> W[旋轉能量 E = 命] M[質量 M] --> D[單發種子 D] F --> OM[轉速 ω = 頻率] M --> I[慣量 I] I --> OM end subgraph BUILD[build 層 · POE2 靈魂] FLAT[加法桶] --> MORE[乘法桶 more] MORE --> CONV[轉換 / 觸發 / keystone] end D -->|base| FLAT OM -->|×頻率| OUT[實際 DPS] CONV --> OUT

本文件範圍：只定義「底層」——物理模型、屬性、稀有度、傷害管線接口、成長與門檻。具體流派、關卡內容、美術不在此列（那是底層鎖定後的下一層）。

1. 肉體層：三個輸入、質量的兩張臉

每一場戰鬥由三個輸入決定：

輸入白話來源
力量 F發射灌進的勁道，動能來源等級（唯一）
體積 V陀螺大小 → 半徑屬性點道具（可洗，見 §7）
質量 M陀螺有多少「料」屬性點道具（可洗，見 §7）

質量的兩張臉：質量 ≠ 重量

質量是唯一輸入，但它有兩張臉，這兩張臉天生綁在一起、不可分割——這正是取捨的來源：

臉物理意義遊戲效果
質量臉（攻擊/慣性）有多少料 → 動量、轉動慣量單發傷害 D↑、抗擊退↑、慣量 I↑（儲能久）
重量臉（地面效應）料被重力往下壓的力 → 對地壓力抓地/抗出界↑、但地面摩擦↑ → 旋轉能量掉更快
主張：不做「重量」獨立屬性。固定重力下重量 = 質量 × 常數，做兩個屬性只是多記一個沒有獨立決策的數字。堆質量 → 自動同時拿到「撞得痛、推不動、儲能久」與付出「掉速快、要靠抓地補」。物理誠實，且取捨天然成立。

真實佐證：beyblade 分攻擊/防禦/持久型，差別不只總質量，更在「質量分布」（料在邊緣 vs 中心）→ 慣量不同。本設計用體積 V 與質量 M 的組合近似分布（見 §2 的 I 公式）。

2. 轉速 vs 旋轉能量（兩個不同的東西）

物理鐵律：旋轉能量 E = ½ × 轉動慣量 I × 轉速²。轉速與能量是兩回事，分開後系統更好玩：

# 派生量
I = M × V^(2/3)          # 轉動慣量 ∝ 質量 × 半徑²（半徑 ∝ 體積開三次方）

# 旋轉能量 = 命（會消耗、要補充）
E                        # 進場初值由 F 決定；每 tick 被摩擦與碰撞扣

# 轉速 = 由「當下剩多少能量」算出來（不是固定值）
ω = sqrt( 2E / I )       # 能量滿 → 轉得快；能量見底 → 自然慢下來、失穩

# 三個戰鬥表現
攻擊頻率  ∝ ω            # 轉速就是攻擊快慢
單發種子  D = M          # 只看質量 —— 力量再高單發也不變痛
抗擊退    ∝ M            # 越重越站得穩
好處：失穩是免費送的。能量快見底 → ω 自然下降 → 攻擊變慢、開始搖晃、最後停轉。不需要額外寫「失穩規則」，它是物理的自然結果。而且「持久型 = 邊緣配重 = 高慣量 = 轉得久」這種真實流派，直接從公式長出來。
✅ 已定案 A：採物理版。轉速隨能量衰減（ω 由 E 算出），失穩免費送，放棄簡化版（ω=F/I 固定）。代價是平衡較難調——用 §10 的「地面摩擦」與「Flux 回充率」兩個旋鈕，壓制「重陀螺慢轉存超多能量」的物理副作用。

3. 兩池能量迴圈（戰鬥的心臟）

真實陀螺只會掉速、不會自己回充——「能量補充」是我們加的奇幻層，說法藏在 Drive Core（驅動核心）：它是陀螺內建的小馬達/飛輪。做成兩池：

能量池是什麼消耗補充
旋轉能量 E（命）血量 + 轉速來源摩擦持續掉 + 被撞掉靠 Flux 灌回
核心能量 Flux（馬達電）內建馬達的電放主動技能 / 灌回旋轉持續回充（共振 Resonance）+ 撞擊回充 + 擊殺回充
flowchart TD L[發射力量 F] -->|進場| E[旋轉能量 E · 命] FR[共振 Resonance] -->|每秒回充| FX[核心能量 Flux] HIT[撞到人 / 擊殺] -->|跳電| FX E -->|摩擦 + 被撞| DR[持續流失] FX -->|灌回旋轉| E FX -->|放主動技能| SK[爆發輸出] DR --> Q{Flux 還夠嗎?} Q -->|夠| E Q -->|乾了| DEAD[E 只掉不回
死亡螺旋 → 停轉]
藏在裡面的核心決策：Flux 有限，你要花在「灌回旋轉（苟活）」還是「放技能（爆發打人）」？激進派用 Flux 轟人、靠撞擊回充滾雪球；穩健派用 Flux 續命、磨死對手。這一個選擇就撐起兩種流派，而且回充（共振）是可以刷的 POE 詞綴，直接咬回 build 層。

4. 三圍（剛 / 旋 / 蓄）＝ 門檻 + 升級成長

⚠️ 本節已作廢（2026-07-02 定案）：砍掉三圍。兩輪獨立分析發現剛/旋/蓄與物理量並非 1:1（旋=ω 是動態衍生值、蓄=能量是衍生容量、只有剛=配重是原生），別名會讓玩家誤以為是三條可獨立加點的屬性＝製造謊言。改為：門檻與顯示一律直接用誠實物理名（配重 / 體積 / 旋能 / 核心能量 / 轉速），零翻譯層。以下三圍內容僅存查，實作依 docs/15。

對應 POE2 的 力/敏/智。一個系統身兼兩職：既是升級的成長物，也是裝備/技能的門檻（＝「白裝」機制的判定來源）。

陀螺三圍對應 POE2管什麼 / 當誰的門檻
剛STR 力重量、撞擊 —— 重配重盤、轟擊 Drive 的資格
旋DEX 敏轉速、抓地 —— 高速軸尖、多段 Drive 的資格
蓄INT 智能量、回充 —— 能量爆發 Drive、共振裝的資格
  • 等級：給力量 F（驅動 ω 與 E＝「量」），並依陀螺類型（職業）少量偏向地加三圍（照 POE2 職業偏向）。
  • 屬性點道具（可洗）：讓玩家自己加碼三圍或體積/質量，決定要達成哪些門檻。
  • 三圍作用：① 當裝備/Drive 的門檻（＝白裝機制）② 提供少量被動加成。
✅ 已定案 C：三圍給「很少」的加成，主要當門檻。三圍本身只提供極小的被動加成（點綴級，不是傷害來源）。真正的傷害成長來自「天賦點 × 裝備」的連動——玩家靠天賦樹的乘算節點與裝備詞綴湊出流派，而不是靠堆三圍變強。三圍只負責「開資格」與微幅調味。

稀有度階梯（比照 POE2）

稀有度決定「有幾條詞綴」，掉落樓層（item level）決定「詞綴能滾到多好的階（T1 最好）」。

6. 傷害公式管線 × 稀有度（越後段越稀有）

一發傷害從「種子」被層層放大的生產線。越靠後段（乘法、轉換、質變）越稀有；越靠前段（加法）越普通。

肉體層                         build 層 (resolveTopHit, topDamage.ts)
────────                       ───────────────────────────────────────
M ─► D=M ─► base ─────────┐
                          ├─► conversion   ← attackRing/seal 轉換詞綴（稀有 T3）
flat 詞綴（普通 T1）───────┤
                          ▼
             (base + Σflat)                  ← 加法（白/藍裝，豐富）
             × (1 + Σincreased − Σreduced)   ← 加法桶（藍/黃裝，會鈍化）
             × Π(1+more) × Π(1−less)         ← 乘法桶 ← 只從符文/keystone（稀有 T3）
                          ▼
             mitigate( guard / resist / 敵護甲 A: max(D−A,0) )
             × hitChance × critEV            ← attackRing 暴擊（稀有 T2–T3）
                          ▼
             單發傷害 perHit  ── 這是「質」

ω(由 E 算出) ──► × 攻擊頻率 ──► DPS = perHit × freq(ω)  ── 這是「量」
E ──► 血量（每次被扣對方 perHit）

算例：親眼看到每個屬性怎麼進乘算

步驟重擊流（M=8）高速流（M=2）
① 種子 D=M82
② +flat（+2）104
③ ×(1+increased 40%)145.6
④ ×more 60%（重擊 Drive）22.4—（速度流不帶 more）
⑤ 暴擊 EV ×1.2≈ 26.9≈ 6.7
頻率（ω）慢（每秒約 0.4 下）快（每秒約 1.6 下）
護甲前 DPS≈ 10.6≈ 10.7（量補質，打平）
對護甲 A=626.9−6 = 20.9 戳穿6.7−6 = 0.7 幾乎戳不動
證明使用者要的事成立：護甲前兩者 DPS 打平（等級買量不買質），護甲後重陀螺完勝（質破牆）。每個屬性——M 進 base、詞綴進 increased、Drive 進 more、ω 進頻率、護甲進 mitigate——都在乘算鏈上有各自明確的位置。

7. 技能系統與洗點（比照 POE2）

組合規則

  • 一顆陀螺帶 1 個主動 Drive + 最多 5 個輔助 Rune（＝ 6-link）；輔助插槽數隨進度 2 → 3 → 5。
  • more 乘法只從 Rune（＝輔助寶石）來，稀缺性靠兩條：① 每顆 Drive 的插槽有限、② 每顆符文整台陀螺只能用一次（POE2 規則）。repo supportFamily 已防同家族疊加，補一條「全 build 唯一」即對齊。

「白裝」＝ 裝了等於沒裝：三層發動門檻

條件層POE2 做法套進陀螺
① 裝備資格三圍不夠根本裝不上剛/旋/蓄 不足 → Drive 裝不上
② 標籤相容近戰輔助不能掛法術符文 tag 要對得上 Drive（isRuneCompatible 已做）
③ 發動條件連段/觸發前置質量<門檻的重擊 Drive = 裝了也打不出 = 白裝
✅ 已定案 B：硬門檻（比照 POE2）。第 ③ 層條件不到 → Drive 完全不發動（裝了等於沒裝），不做打折。流派邊界清楚，湊條件的成就感明確。

洗點

屬性點可洗，付代價（覆蓋交接文件「不可洗」）。比照 POE2：花遊戲金幣類資源洗，前期便宜、後期越貴。用「成本」讓決策有重量，而非「不可逆」——既能自由試流派、配錯不廢檔，又不會變成無重量的隨意切換。

8. 兩層接口速查（動工對照卡）

肉體層屬性進公式哪一格身分
質量 M① base 種子（D=M）「質」——所有倍數乘它身上
力量 F不進單發；驅動 ω 與 E「量」的引擎
轉速 ω（由 E 算）頻率（乘在總量外）攻擊快慢
旋轉能量 E血量存活池
體積 V進 I 壓低 ω + 開放 AoE + 裝備門檻取捨旋鈕
接口關鍵一刀（現有 code bug）：arenaRuntime.ts / topCombat.ts 目前 packet.impact = impact + mass × rpm × 18，把質量與轉速都灌進單發 = 雙重灌水，違反「D=M、ω=頻率」。改成：per-hit base = M × κ（只吃質量），rpm 移出 per-hit、改乘在頻率項。這一刀拆開，兩層才無縫咬合。

9. 爬塔節奏（承接交接文件，維持不變）

F(L)   = 10 × L^0.7        # 等級→力量：次線性，邊際遞減，逼玩家轉養成
Req(n) = 50 × 1.15^n       # 塔層難度：每層 +15% 複利
有效傷害 = max(D − A, 0)    # 後期敵人護甲 A

複利成長終究甩開次線性 → 純刷等級必然撞牆（設計目標）。純刷級推塔：10 層≈4 級、20≈10、30≈27、40≈73（軟牆）、50≈200（自殺）。卡關 → 換重陀螺/點乘算技能破牆 → 更高層再卡 → 再養成，這個循環是遊戲的心跳。

11. 設計定案（三個岔路已鎖定）

A. 轉速 → 物理版（已定案）。ω 由能量 E 衰減算出，失穩免費。用 §10 摩擦/回充旋鈕平衡「慢轉存超多能量」的副作用。
B. 白裝門檻 → 硬（已定案，比照 POE2）。條件不到，Drive 完全不發動，不打折。
C. 三圍 → 少量加成、主要當門檻（已定案）。三圍只點綴，主要傷害來自「天賦點 × 裝備」連動。
建議動工順序（底層鎖定後）：① 拆開「雙重灌水」接口一刀（§8）② 接通兩池能量迴圈（§3，讓玩家會輸）③ 質量/慣量/能量衍生公式（§2）④ 稀有度 + 前綴後綴詞綴池（§5）⑤ 條件門檻系統（§4、§7 白裝）⑥ more 稀缺化 + keystone（§6、§7）。這一步才建議動用 Fable 出正式實作計畫、派 Sonnet 執行。
